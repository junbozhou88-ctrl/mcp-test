# ZenFlow 产品需求文档

## 1. 产品概述

ZenFlow 是一款面向移动端的极简任务与专注管理应用。产品目标是帮助用户以更平和、低干扰的方式管理每日待办、设置提醒、回顾完成情况，并通过连续专注数据形成长期正反馈。

当前版本基于 Stitch 原型「Creative Todo Canvas」中的移动端设计稿实现，覆盖首页、全部任务、新增任务、编辑任务、任务统计、个人中心 6 个核心页面。

## 2. 产品定位

- 产品名称：ZenFlow / 禅流
- 产品类型：移动端任务管理与专注回顾工具
- 目标用户：需要轻量待办管理、偏好克制视觉风格、希望降低任务焦虑的个人用户
- 核心价值：以低噪音界面承载任务管理，让用户聚焦当下任务、保持节奏、持续复盘

## 3. 设计原则

- 纯粹：页面信息密度可控，不使用复杂仪表盘干扰用户。
- 平静：深色背景、柔和绿色强调色、少量动效，营造稳定专注感。
- 快速：常用操作不超过两步，包括查看任务、新增任务、编辑任务。
- 连续：通过统计与个人中心强化完成率、连续天数等长期反馈。

## 4. 信息架构

当前项目使用轻量 hash route 实现多页面路由。

| 路由 | 页面 | 来源原型 | 说明 |
| --- | --- | --- | --- |
| `#/` | 首页 | 禅流移动端 - 纯粹任务版 | 展示今日概要与今日任务 |
| `#/tasks` | 全部任务 | 禅流 - 全部任务 | 按时间分组展示全部待办 |
| `#/new` | 新增任务 | 禅流 - 新增任务 | 创建新任务并设置提醒时间 |
| `#/edit` | 编辑任务 | 禅流 - 编辑任务 | 修改任务标题、提醒时间或删除 |
| `#/stats` | 任务统计 | 禅流 - 任务统计 | 展示周完成数据与趋势 |
| `#/profile` | 个人中心 | 禅流 - 极简个人中心 | 展示用户信息、连续专注与设置入口 |

## 5. 用户流程

### 5.1 查看今日任务

1. 用户进入首页。
2. 查看今日概要，包括剩余任务数与完成进度。
3. 浏览今日任务卡片。
4. 点击任务卡进入编辑任务页。

### 5.2 查看全部任务

1. 用户在首页点击「查看全部」。
2. 进入全部任务页。
3. 任务按「今天」「明天」「稍后」分组展示。
4. 用户点击任意任务进入编辑任务页。

### 5.3 新增任务

1. 用户点击首页右下角新增按钮。
2. 进入新增任务页。
3. 输入任务标题。
4. 设置提醒时间。
5. 点击「添加任务」后返回全部任务页。

### 5.4 编辑任务

1. 用户从首页或全部任务页点击任务。
2. 进入编辑任务页。
3. 修改任务标题或提醒时间。
4. 点击「保存修改」返回全部任务页。
5. 如需移除任务，点击「删除任务」。

### 5.5 查看统计

1. 用户点击底部导航中的统计入口。
2. 查看本周完成任务数、完成率、连续达成数据。
3. 查看过去 7 天完成趋势与任务分类占比。

### 5.6 查看个人中心

1. 用户点击头像或底部导航中的个人入口。
2. 查看用户昵称、等级、连续专注天数。
3. 进入账号安全或帮助反馈入口。

## 6. 页面需求

### 6.1 首页

首页用于承载用户进入应用后的首要任务视图。

功能需求：
- 展示品牌标识 ZenFlow。
- 展示用户头像入口。
- 展示今日概要卡片，包括剩余任务数与完成进度。
- 展示引导文案，降低任务焦虑。
- 展示今日任务列表。
- 支持点击「查看全部」进入全部任务页。
- 支持点击任务进入编辑任务页。
- 支持点击新增按钮进入新增任务页。
- 支持底部导航切换首页、统计、个人中心。

验收标准：
- 首页首屏能看到品牌、今日概要、今日任务标题和至少 2 条任务。
- 点击「查看全部」跳转 `#/tasks`。
- 点击新增按钮跳转 `#/new`。
- 点击任务卡跳转 `#/edit`。

### 6.2 全部任务页

全部任务页用于集中浏览待办。

功能需求：
- 顶部展示返回按钮和页面标题「全部任务」。
- 展示待办总数。
- 提供「时间倒序」排序入口。
- 按时间分组展示任务。
- 每条任务展示完成状态、标题、时间和进入详情箭头。
- 点击任务进入编辑任务页。

验收标准：
- 页面至少展示「今天」「明天」「稍后」三个分组。
- 点击返回按钮回到首页。
- 点击任意任务跳转 `#/edit`。

### 6.3 新增任务页

新增任务页用于快速创建待办。

功能需求：
- 提供关闭按钮，返回首页。
- 提供任务标题输入区域。
- 提供提醒时间入口。
- 提供主操作按钮「添加任务」。
- 添加成功后返回全部任务页。

验收标准：
- 输入框 placeholder 为「书写此刻的任务...」。
- 默认提醒时间展示为「今天 09:00」。
- 点击「添加任务」跳转 `#/tasks`。

### 6.4 编辑任务页

编辑任务页用于修改已有任务。

功能需求：
- 提供关闭按钮，返回首页。
- 展示当前任务标题。
- 支持修改任务标题。
- 支持查看或修改提醒时间。
- 提供主操作按钮「保存修改」。
- 提供删除任务入口。

验收标准：
- 默认任务标题为「重塑产品交互体验设计」。
- 默认提醒时间展示为「今天 14:00」。
- 点击「保存修改」跳转 `#/tasks`。
- 页面可见「删除任务」入口。

### 6.5 任务统计页

任务统计页用于任务完成回顾。

功能需求：
- 展示品牌与头像入口。
- 展示本周完成任务总数。
- 展示效率提升描述。
- 展示完成率和连续达成数据。
- 展示过去 7 天完成趋势图。
- 展示任务分类占比。
- 支持底部导航切换页面。

验收标准：
- 页面主标题为「本周完成：42个任务」。
- 可见完成率 `84%` 与连续达成 `45`。
- 可见趋势图与任务分类卡片。

### 6.6 个人中心页

个人中心页用于承载用户身份与设置入口。

功能需求：
- 展示品牌与头像。
- 展示用户头像、昵称、等级。
- 展示连续专注天数。
- 展示账号与安全入口。
- 展示帮助与反馈入口。
- 展示版本号。
- 支持底部导航切换页面。

验收标准：
- 用户昵称展示为「林深」。
- 连续专注天数展示为 `24`。
- 可见「账号与安全」「帮助与反馈」两个入口。

## 7. 交互规则

### 7.1 路由与导航

- 应用使用 hash route，在不刷新页面的情况下完成页面切换。
- `#/` 为默认首页；未知路由回落到首页。
- 底部导航固定在页面底部，包含：首页、统计、个人中心。
- 底部导航只在首页、全部任务、统计、个人中心展示；新增和编辑页不展示底部导航，避免干扰提交操作。
- 首页新增按钮固定在底部导航上方，点击进入 `#/new`。
- 顶部品牌点击回到首页；头像点击进入个人中心。

### 7.2 任务浏览

- 首页只展示「今天」分组中的前 3 条任务。
- 首页今日概要中的剩余任务数根据「今天」分组未完成任务动态计算。
- 全部任务页按「今天」「明天」「稍后」分组展示任务。
- 全部任务页的任务数量根据当前任务列表动态计算。
- 点击任务文字区域进入编辑页，路由格式为 `#/edit/:taskId`。
- 点击任务完成圆点只切换完成状态，不进入编辑页。

### 7.3 新增任务

- 新增任务页默认标题为空，默认提醒时间为 `09:00`，默认分组为「今天」，默认分类为「重点任务」。
- 标题为空时，「添加任务」按钮禁用。
- 用户可选择提醒时间、任务分组和任务分类。
- 点击「添加任务」后，将任务写入当前前端状态，并跳转到 `#/tasks`。
- 新增任务默认状态为未完成。

### 7.4 编辑任务

- 编辑页根据路由中的 `taskId` 读取任务。
- 用户可修改标题、提醒时间、任务分组和任务分类。
- 标题为空时，「保存修改」按钮禁用。
- 点击「保存修改」后更新当前前端状态，并跳转到 `#/tasks`。
- 点击「删除任务」后从当前前端状态中移除任务，并跳转到 `#/tasks`。
- 点击关闭按钮放弃本次编辑并返回首页。

### 7.5 排序

- 全部任务页提供时间排序按钮。
- 点击排序按钮在「时间正序」与「时间倒序」之间切换。
- 排序只影响每个分组内部的任务排列，不改变「今天」「明天」「稍后」的分组顺序。

### 7.6 统计

- 统计页的完成率由当前任务列表实时计算。
- 本周完成数基于当前完成任务数动态更新。
- 完成状态切换、新增、删除任务后，统计页数据应随状态变化刷新。

### 7.7 页面布局

- 页面在移动端宽度下占满屏幕。
- 桌面预览环境中展示为手机画布。
- 底部导航和新增按钮不能互相遮挡。
- 关键操作按钮需要保持足够点击区域，适配单手操作。

## 8. 数据模型

### 8.1 Task

```ts
type Task = {
  id: string
  title: string
  time: string
  dateGroup: '今天' | '明天' | '稍后'

```

type User = {
  name: string

### 8.3 Stats

```ts
type Stats = {
  weeklyCompleted: number
  completionRate: number
  streakCount: number
  categoryRatios: {
    focus: number
    creative: number
    review: number
  }

```

## 9. 非功能需求

- 响应式：桌面展示手机原型壳，移动端全屏展示。
- 可访问性：主要按钮提供可读文本或 `aria-label`。
- 性能：首屏为纯 React + CSS，无额外路由依赖。
- 可维护性：后续可将页面组件拆分到 `src/pages`，将导航和头部拆分到 `src/components`。
- 兼容性：支持现代 Chromium、Safari、Firefox。
